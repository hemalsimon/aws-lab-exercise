
# 🔐 AWS IAM Lab Exercise (End-to-End)

## 🎯 Objective

* Create IAM user
* Enable MFA
* Test **no permissions**
* Create group & assign permissions
* Validate EC2 + S3 access
* Configure AWS CLI
* Perform actions via CLI

---

# 🧪 Step 1: Login as Root User

👉 Go to: AWS Management Console
Login using **root account**

---

# 🏷️ Step 2: Create Account Alias

1. Go to **IAM → Dashboard**
2. Click **Create account alias**
3. Example:

```
my-aws-lab-account
```

👉 Login URL becomes:

```
https://my-aws-lab-account.signin.aws.amazon.com/console
```

---

# 👤 Step 3: Create IAM User

1. Go to **IAM → Users → Create user**
2. Username:

```
lab-user
```

3. Select:

* ✅ AWS Management Console access
* Password: Custom

4. DO NOT attach permissions (important for test)

---

# 🔐 Step 4: Configure MFA

1. Open user → **Security credentials**
2. Assign MFA
3. Use:

* Google Authenticator or
* Microsoft Authenticator

---

# 🔍 Step 5: Login as IAM User & Test (No Permission)

Login using alias URL

👉 Try:

* EC2 → List instances
* Try create EC2
* Try S3 access

✅ Expected result:

```
Access Denied
```

---

# 👥 Step 6: Create Admin Group

1. IAM → Groups → Create group
2. Name:

```
admin-group
```

3. Attach policy:

* AdministratorAccess

---

# ➕ Step 7: Add User to Group

* Add **lab-user → admin-group**

---

# 🔄 Step 8: Login Again & Validate Access

Login again as `lab-user`

👉 Now test:

* Create EC2 instance
* List EC2
* Create S3 bucket

✅ Expected:
✔ Full access works

---

# 🔑 Step 9: Enable Programmatic Access (CLI)

1. IAM → Users → lab-user
2. Go to **Security credentials**
3. Create **Access Key**

Save:

```
Access Key ID
Secret Access Key
```

---

# 💻 Step 10: Configure AWS CLI

Install CLI:
👉 AWS CLI

### Run:

```bash
aws configure
```

Enter:

```
Access Key ID:
Secret Access Key:
Region: ap-south-1
Output: json
```

---

# 👤 Step 11: Check Logged-in User

```bash
aws sts get-caller-identity
```

✅ Output shows:

```
UserId
Account
Arn
```

---

# ☁️ Step 12: EC2 Commands via CLI

## List EC2 instances

```bash
aws ec2 describe-instances
```

---

## Create EC2 instance

```bash
aws ec2 run-instances \
  --image-id ami-0f58b397bc5c1f2e8 \
  --count 1 \
  --instance-type t2.micro \
  --key-name my-key \
  --security-group-ids sg-xxxx \
  --subnet-id subnet-xxxx
```

---

## Terminate EC2

```bash
aws ec2 terminate-instances --instance-ids i-xxxx
```

---

# 🪣 Step 13: S3 Commands

## List buckets

```bash
aws s3 ls
```

## Create bucket

```bash
aws s3 mb s3://my-lab-bucket-12345
```

---

# ✅ Validation Summary

| Step           | Expected Result |
| -------------- | --------------- |
| Before group   | ❌ Access denied |
| After group    | ✅ Full access   |
| CLI configured | ✅ Commands work |

---

# 🧠 Bonus (Best Practice)

* Never use root user
* Always enable MFA
* Use **groups for permissions**
* Use **least privilege policy** instead of admin in real projects
