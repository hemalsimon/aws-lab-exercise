

# 🔐 AWS IAM Lab Exercise (Step-by-Step)

## 🎯 Goal

* IAM user (no permission → test deny)
* MFA setup
* Group + permissions (EC2 + S3 / Admin)
* Validate access
* Configure CLI
* Perform EC2 + S3 operations

---

# 🧪 Step 1: Login as Root

Go to 👉 AWS Management Console
Login with root user

---

# 🏷️ Step 2: Create Account Alias

IAM → Dashboard → **Create account alias**

Example:

```
my-lab-account
```

Login URL:

```
https://my-lab-account.signin.aws.amazon.com/console
```

---

# 👤 Step 3: Create IAM User

IAM → Users → Create user

* Username: `lab-user`
* Access:

  * ✅ Console access
  * Password: custom
* ❌ Do NOT assign permissions

---

# 🔐 Step 4: Configure MFA

IAM → Users → lab-user → Security credentials → Assign MFA

Use:

* Google Authenticator
  or
* Microsoft Authenticator

---

# ❌ Step 5: Login & Validate NO Permission

Login using alias URL with `lab-user`

Try:

* EC2 → List instances
* Create EC2
* Delete EC2

✅ Expected:

```
Access Denied
```

---

# 👥 Step 6: Create Group (Admin)

IAM → Groups → Create group

* Name: `admin-group`
* Attach policy:

  * AdministratorAccess
    OR
  * AmazonEC2FullAccess
  * AmazonS3FullAccess

---

# ➕ Step 7: Add User to Group

* Add `lab-user` → `admin-group`

---

# 🔄 Step 8: Login Again & Validate Access

Login again as `lab-user`

Now test:

### EC2

* List instances
* Create instance
* Terminate instance

### S3

* Create bucket
* List buckets

✅ Expected:
✔ All actions allowed

---

# 🔑 Step 9: Enable CLI Access

IAM → Users → `lab-user` → Security credentials

Create:

* Access Key ID
* Secret Access Key

---

# 💻 Step 10: Configure AWS CLI

Install 👉 AWS CLI

Run:

```bash
aws configure
```

Enter:

```
Access Key ID:
Secret Access Key:
Region: ap-south-1
Output: json
```

---

# 👤 Step 11: Check Logged-in User

```bash
aws sts get-caller-identity
```

✅ Output:

* Account
* ARN
* User

---

# 🔍 Step 12: Check Permissions

Try a restricted action (before adding group you would get denied, now it should work):

```bash
aws ec2 describe-instances
aws s3 ls
```

---

# ☁️ Step 13: EC2 via CLI

## List instances

```bash
aws ec2 describe-instances
```

## Create instance

```bash
aws ec2 run-instances \
  --image-id ami-0f58b397bc5c1f2e8 \
  --count 1 \
  --instance-type t2.micro
```

## Terminate instance

```bash
aws ec2 terminate-instances --instance-ids i-xxxx
```

---

# 🪣 Step 14: S3 via CLI

## List buckets

```bash
aws s3 ls
```

## Create bucket

```bash
aws s3 mb s3://my-unique-lab-bucket-12345
```

---

# ✅ Final Validation Checklist

| Test               | Result           |
| ------------------ | ---------------- |
| Before permissions | ❌ Access denied  |
| After group added  | ✅ Access allowed |
| MFA enabled        | ✅ Secure login   |
| CLI configured     | ✅ Working        |
| EC2 via CLI        | ✅ Created/List   |
| S3 via CLI         | ✅ Created/List   |

---

# 🧠 Important Interview Points

* IAM user has **no permissions by default**
* Use **groups for permission management**
* Enable **MFA for security**
* Use **CLI for automation**
* Prefer **least privilege** over admin access
